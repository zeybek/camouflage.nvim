# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in camouflage.nvim, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Email the maintainer directly or use [GitHub's private vulnerability reporting](https://github.com/zeybek/camouflage.nvim/security/advisories/new)
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Fix & Disclosure**: Coordinated based on severity

## Scope

This security policy applies to:
- The camouflage.nvim plugin code
- Dependencies used by the plugin
- GitHub Actions workflows

## Security Best Practices

When using camouflage.nvim:
- Always use the latest version
- Review `.camouflage.yaml` project configs before trusting them
- Be cautious with custom patterns that may expose sensitive data
